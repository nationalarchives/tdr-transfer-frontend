Long File Path Diagnostic Test

This page tests whether your browser can correctly read files with paths longer than 260 characters. It checks file.size, file.stream(), and computes the SHA-256 checksum.

Test 1: Direct long path

Instructions:

  1. Create a folder with a path longer than 260 characters (e.g. deeply nested folders)
  2. Place a file with known content inside it
  3. Either drag & drop the folder below, or use the file picker

Test 2: Using a symlink/junction (workaround)

This test demonstrates that using a junction preserves the internal folder structure while shortening the root path.

  1. Open Command Prompt as Administrator
  2. Create a junction to your long path: 
    mklink /J C:\shortlink "C:\Users\name\Very\Long\Department\Name\Project\SubFolder\MoreFolders"
  3. Now drop or select C:\shortlink below
  4. Compare the file paths — the internal folder structure should be identical, just with a shorter root
